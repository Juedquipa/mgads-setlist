1. Resumen Ejecutivo
Neon Beats SETLIST es una aplicación móvil Android que digitaliza y democratiza la experiencia musical dentro de bares y establecimientos de entretenimiento. Permite a los clientes solicitar canciones desde su mesa escaneando un código QR, sin necesidad de registro, mediante un sistema de créditos vinculados al consumo activados por el mesero.
El sistema integra tres roles diferenciados — Cliente, Mesero y Administrador/DJ — con flujos de usuario independientes, gestión de cola de reproducción en tiempo real y control de calidad del contenido musical mediante un proceso de aprobación administrado por el DJ. La integración con la API pública de Spotify garantiza un catálogo musical profesional con metadatos, previsualizaciones y búsqueda avanzada.

2. Problemática Identificada y Justificación
2.1 Problemática
En los bares y establecimientos de entretenimiento con música en vivo o ambiente musical controlado por un DJ, la experiencia del cliente frente a la música es completamente pasiva. Los mecanismos actuales para solicitar canciones son informales, ineficientes y generan fricciones:
El cliente debe levantarse de su mesa y acercarse físicamente al DJ para hacer una solicitud, interrumpiendo su experiencia social.
Las solicitudes se hacen de forma verbal, sin registro ni control, generando confusión, olvidos y favoritismos involuntarios.
No existe equidad entre mesas: una mesa puede monopolizar la selección musical mientras otras no participan.
El DJ no tiene herramientas para filtrar contenido inapropiado ni controlar la calidad del material propuesto por los clientes.
El bar no cuenta con mecanismos para vincular la participación musical al consumo, desaprovechando una palanca de fidelización.
2.2 Justificación
La música es uno de los factores más determinantes en la experiencia dentro de un bar. Neon Beats SETLIST propone una solución tecnológica que digitaliza y democratiza la experiencia musical mediante:
Acceso sin fricción: el cliente accede desde su propio dispositivo móvil escaneando un QR, sin registro previo.
Sistema de créditos vinculado al consumo: cada pedido activa créditos de solicitud musical, incentivando directamente el consumo.
Control de calidad: todas las canciones externas pasan por aprobación del admin antes de ingresar a la cola.
Equidad entre mesas: el sistema de créditos garantiza que todas las mesas participen en igualdad de condiciones.
Visibilidad en tiempo real: clientes y DJ comparten la misma información sobre la cola de reproducción.
3. Objetivos del Proyecto
3.1 Objetivo General
Desarrollar una aplicación móvil Android funcional que permita a los clientes de un bar solicitar canciones desde su mesa mediante un sistema de créditos vinculados al consumo, integrando roles diferenciados para clientes, meseros y administradores, con gestión en tiempo real de la cola de reproducción musical y búsqueda a través de la API pública de Spotify.
3.2 Objetivos Específicos
Diseñar e implementar una interfaz móvil de alta fidelidad con tres flujos de usuario diferenciados (Cliente, Mesero, Administrador/DJ) siguiendo principios de UI/UX centrado en el usuario, con identidad visual Industrial Neon Underground.
Implementar un sistema de autenticación por roles: acceso del administrador y mesero mediante credenciales JWT, y acceso del cliente mediante escaneo de código QR de mesa con ML Kit.
Desarrollar el mecanismo de activación de créditos musicales mediante códigos PIN de 6 dígitos generados por el mesero, con tiempo de expiración configurable y validación en tiempo real contra el backend.
Construir un módulo de gestión de cola de reproducción que permita al DJ visualizar, reordenar, aprobar y rechazar canciones en tiempo real con actualización automática en todos los dispositivos conectados.
Integrar la API pública de Spotify para búsqueda de canciones, recuperación de metadatos (portada, artista, duración, preview de 30 segundos) y reproducción del catálogo del bar.
Implementar la persistencia de datos mediante arquitectura MVVM con Room para caché local y backend REST propio (Node.js + PostgreSQL) con despliegue en plataforma cloud (Railway o Render).
Distribuir la aplicación de forma instalable mediante Firebase App Distribution y contenerizar el backend con Docker para despliegue reproducible.

5. Integración con la API Pública de Spotify
5.1 Descripción General
Neon Beats SETLIST utiliza la Spotify Web API (API pública gratuita) para proveer un catálogo musical profesional con metadatos completos, imágenes de alta calidad, previsualizaciones de audio y búsqueda avanzada. Esta integración reemplaza la necesidad de mantener un catálogo propio de metadatos musicales.

5.3 Autenticación con Spotify
Se utiliza el flujo Client Credentials (OAuth 2.0) que no requiere autenticación del usuario final, ideal para acceso al catálogo público:
El backend obtiene un access_token usando client_id y client_secret registrados en Spotify Developer Dashboard.
El token se renueva automáticamente cada 3600 segundos (1 hora).
El cliente Android nunca maneja credenciales de Spotify — todas las llamadas se proxean a través del backend propio para seguridad.

5.5 Catálogo del Bar
El administrador construye el catálogo propio del establecimiento buscando canciones a través de Spotify y guardando sus IDs en la base de datos propia. Esto permite:
Búsqueda filtrada: los clientes solo ven el catálogo aprobado por el bar, no todo Spotify.
Metadatos siempre actualizados: se recuperan de Spotify en tiempo real, sin mantenimiento manual.
Propuestas externas: los clientes pueden proponer canciones fuera del catálogo, que el DJ evalúa usando el preview de Spotify antes de aprobar o rechazar.
6. Arquitectura del Sistema
6.1 Visión General
El sistema sigue una arquitectura cliente-servidor con tres capas bien definidas: aplicación Android (cliente), backend REST propio (servidor) y servicios externos (Spotify API). La app Android implementa el patrón MVVM obligatorio con Repository Pattern para abstracción de fuentes de datos.

15. Glosario
Término
Definición
Mesa
Identificación física del grupo de clientes, activada por escaneo de QR único por mesa
QR de mesa
Código QR impreso en la mesa que identifica al cliente en el sistema al ser escaneado
Solicitud
Petición de reproducción de una canción realizada por un cliente desde su mesa
Crédito
Unidad de solicitud disponible, otorgada al cliente mediante código PIN del mesero
Código / PIN
Número único de 6 dígitos generado por el mesero que activa créditos de canciones por pedido
Catálogo del bar
Biblioteca de canciones de Spotify pre-aprobadas por el administrador para solicitud directa
Cola
Lista ordenada y priorizada de canciones próximas a reproducirse, gestionada por el DJ
Admin / DJ
Mismo rol — usuario con acceso completo al sistema, responsable de la música del bar
Mesero
Usuario con acceso limitado — genera códigos PIN por pedido durante su turno
MVVM
Model-View-ViewModel — arquitectura de software obligatoria para el proyecto
Room
Librería de persistencia local SQLite para Android, usada para caché offline
Retrofit
Cliente HTTP para Android usado para consumir la API REST propia y Spotify
Composable
Función de Jetpack Compose que define un elemento de UI declarativo
JWT
JSON Web Token — estándar de autenticación sin estado para admin y mesero
Spotify Web API
API pública de Spotify para búsqueda y metadatos de canciones (portada, artista, duración, preview)
ML Kit
Librería de Google para procesamiento de imágenes en Android, usada para escaneo de QR
